# E501 — E396 warm-start on the E500 corpus

E501 tests whether the durable, structurally stronger E396 checkpoint benefits
from the clean E500 documentized-expression corpus. It also closes a harness
gap: bit-exact `--resume-from` correctly rejects a changed data manifest, but
there was no explicit way to initialize compatible weights for a new corpus.

## Harness intervention

`train_model --initialize-from <last.pt>` now loads compatible serving weights
and tokenizers before optimizer creation while resetting optimizer state, RNG,
step, and token counters. It is mutually exclusive with `--resume-from`; the
existing resume data/mixture guards remain unchanged. Focused tests prove exact
weight preservation at step zero, counter reset, new-corpus admission, and the
mutual-exclusion error. The train harness is stamped `v2`.

## Matched recipe

All arms use CPU, frozen local SmolLM2-135M context, choice output, d128/h4/c2/dn4,
batch 2, learning rate `3e-4`, seed 0, the E396 auxiliary-loss recipe, no DESIGN
context, and the committed 260-row E500 corpus. Every train summary records
`max_wall_minutes=3.0`; every process used an external 170-second cap.

Evaluation is the same honest smoke `n=3` for every arm: prompt-derived slot
contract, constrained decode, no unconstrained fallback, four generation steps,
one attempt, and a 96-token LTR cap. Every evaluation emitted AgentEvals JSONL
and a pinned AgentV bundle without execution errors.

The published mixture intentionally balances the three present task groups,
reducing generation exposure from 94.6% in the corpus to 33.9% in the measured
draws. Uniform 5k restores generation exposure and produces nonzero component
recall, but structure collapses and generated programs show duplicate/nested
subtree spam. The 1k arm preserves the parent's behavior and adds `+0.0200`
structure, but no semantic gate moves.

## Decision

Keep the generalized initialization path: it makes new-corpus continuation
explicit without weakening resume provenance. Reject all E501 checkpoints for
promotion or bucket sync. This is smoke-only diagnostic evidence, not ship
readiness. The next continuation lever must protect the parent against
catastrophic forgetting—lower update magnitude or an explicit retention
objective—before extending beyond 1k target tokens.
