# Preregistered experiment campaigns

## Claim

An experiment result is confirmatory only when execution and reporting bind to
the same decision-complete manifest digest recorded before outcome access.
Promotion is a derived claim over complete locked evidence, never a label
selected by the caller.

## Why it might be true

Frozen registrations prevent outcome-informed plan replacement; append-only
amendments preserve useful exploration without rewriting the confirmatory
question. Prospectively declared Holm families connect the statistical
decision to the exact hypotheses that were planned. See
[[gate-reachability-and-power]] for the separate evidence-volume boundary.

## Falsification boundary

The contract fails if a runner can execute without a pre-start lock, a changed
digest can validate an old result, incomplete arm/seed or artifact evidence can
promote, or an exploratory deviation can replace the registered analysis.

## Status & next step

SLM-337 adds the canonical lock, runner/outcome binding, result validator, and
AP-007 fixture seam. AP-001 must certify meaning-v2 before the primary endpoint
switches from binder/reference F1; AP-007+ runners should consume this contract
rather than define new manifest schemas.
